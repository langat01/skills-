SkillForge Backend GitHub Setup

A complete guide to hosting your backend code on GitHub

How to Set Up Your Backend on GitHub

Yes, you can copy and paste the code to GitHub, but there's a specific process to follow. Here's how:

1. Create a New Repository on GitHub

  • Go to github.com and log in
  • Click the "+" icon in the top right and select "New repository"
  • Name it "skillforge-backend"
  • Add a description: "Backend for SkillForge learning platform"
  • Choose "Public" or "Private" (Public is free)
  • Initialize with a README (important!)
  • Add .gitignore: Select "Node.js"
  • Choose MIT license
  • Click "Create repository"

2. Clone the Repository to Your Computer

git clone https://github.com/your-username/skillforge-backend.git

3. Add Your Backend Files

Copy all the backend files into the cloned repository folder. Your file structure should look like this:

skillforge-backend/ ├── .env ├── .gitignore ├── package.json ├── server.js ├── scripts/ │ └── seedData.js ├── README.md └── LICENSE

Important Note

Make sure to add your MongoDB connection string and JWT secret to the .env file, but do NOT commit sensitive data to GitHub. Use environment variables on your hosting platform instead.

4. Upload Files to GitHub

cd skillforge-backend
git add .
git commit -m "Initial commit: SkillForge backend"
git push origin main

Essential Files for Your Repository

package.json

This file defines your project dependencies and scripts:

{ "name": "skillforge-backend", "version": "1.0.0", "description": "Backend for SkillForge learning platform", "main": "server.js", "scripts": { "start": "node server.js", "dev": "nodemon server.js", "seed": "node scripts/seedData.js" }, "dependencies": { "express": "^4.18.2", "mongoose": "^7.5.0", "bcryptjs": "^2.4.3", "jsonwebtoken": "^9.0.2", "cors": "^2.8.5", "dotenv": "^16.3.1" }, "devDependencies": { "nodemon": "^3.0.1" }, "keywords": ["learning", "courses", "education"], "author": "Your Name", "license": "MIT" }

README.md

This file documents your project for other developers:

# SkillForge Backend Backend API for the SkillForge learning platform built with Node.js, Express, and MongoDB. ## Features - User authentication with JWT - Course management - Community posts and interactions - RESTful API design ## Setup 1. Deploying Your Backend

1. Environment Variables

Create a .env file with your configuration (but don't commit it to GitHub):

MONGODB_URI=mongodb://localhost:27017/skillforge JWT_SECRET=your_super_secret_jwt_key_here_change_in_production JWT_EXPIRES_IN=7d PORT=5000

Security Warning

Never commit your actual .env file to GitHub. It contains sensitive information. Instead, set these environment variables directly on your hosting platform.

2. Popular Deployment Options

  • Heroku: Easy setup with Git integration
  • Railway: Modern platform with GitHub integration
  • Render: Free tier available, simple setup
  • Vercel: Great for frontends, can work with serverless functions
  • DigitalOcean App Platform: Simple and scalable

3. Deploy to Heroku Example

# Create a Procfile for Heroku
web: node server.js
